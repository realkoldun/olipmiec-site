================================================================================
                    1. ВВЕДЕНИЕ
================================================================================

В ходе прохождения практики была выполнена разработка полнофункционального 
веб-приложения для Спортивной школы олимпийского резерва (СДЮШОР) "Олимпиец" 
г. Витебск. Приложение представляет собой современную информационную систему, 
предназначенную для публикации новостей, информации о спортивных секциях, 
тренерском составе и официальных документах школы.

Целью практики являлось приобретение практических навыков разработки 
современных веб-приложений с использованием актуальных технологий и 
фреймворков.

================================================================================
                    2. ОБЩАЯ ХАРАКТЕРИСТИКА ПРОЕКТА
================================================================================

2.1. Назначение приложения

Веб-приложение СДЮШОР "Олимпиец" предназначено для:
- Публикации новостей о спортивных событиях и достижениях школы
- Предоставления информации о спортивных секциях и направлениях подготовки
- Ознакомления с тренерским составом и их квалификацией
- Размещения официальных документов и объявлений
- Обеспечения доступности информации для людей с ограниченными возможностями

2.2. Целевая аудитория

- Учащиеся спортивной школы и их родители
- Тренерский состав и администрация школы
- Посетители и потенциальные клиенты школы
- Люди с ограниченными возможностями зрения

2.3. Основные требования

- Адаптивный дизайн для всех типов устройств
- Поддержка специальных возможностей (accessibility)
- Высокая производительность и скорость загрузки
- SEO-оптимизация
- Безопасность данных

4.3. Ключевые компоненты

4.3.1. Layout компоненты

SiteLayout - основной layout приложения:
- Header - фиксированная шапка с навигацией, поиском и настройками доступности
- Footer - подвал с контактной информацией и ссылками

4.3.2. Компоненты доступности

AccessibilityPanel - панель настроек доступности:
- Изменение размера шрифта (12-24px)
- Режимы контраста (normal, high, dark)
- Масштабирование страницы (1-3x)
- Лупа с регулируемым увеличением (2-10x)
- Озвучка текста (Web Speech API)

TextResizer - компонент изменения размера текста
ContrastToggle - переключатель режимов контраста
VoiceReader - компонент озвучки текста

4.3.3. Компоненты поиска

SearchBar - строка поиска с автодополнением
SearchModal - модальное окно поиска (Ctrl+K)
SearchResults - отображение результатов поиска
SearchHighlight - подсветка совпадений

4.3.4. Компоненты новостей

NewsCard - карточка новости
NewsList - список новостей с пагинацией
NewsDetailPageClient - страница детального просмотра новости

4.4. State Management

4.4.1. Zustand Store

useAccessibilityStore - хранение настроек доступности:
- fontSize: размер шрифта
- fontScale: масштаб текста
- contrast: режим контраста
- zoom: масштаб страницы
- magnifierEnabled: лупа
- voiceEnabled: озвучка
- persist: сохранение в localStorage

useSearchStore - состояние поиска:
- query: текущий запрос
- results: результаты поиска
- status: статус выполнения
- history: история поиска
- selectedType: выбранный тип

4.4.2. TanStack Query

Используется для кэширования и синхронизации серверного состояния:
- useNews - получение новостей с пагинацией
- useLatestNews - последние новости
- useNewsById - новость по ID

Конфигурация кэширования:
- staleTime: 5 минут
- gcTime: 30 минут

Алгоритм релевантности:
- Заголовок: 30 баллов за совпадение

5.5. Telegram интеграция

5.5.1. TelegramScraperService

Сервис для парсинга публичных Telegram каналов через веб-версию:

- getChannelPosts(limit) - получение последних сообщений
- parseMessageText(message) - парсинг текста сообщения
- checkChannelAvailability() - проверка доступности канала
- getChannelInfo() - информация о канале

URL для парсинга: https://t.me/s/{channelId}

Парсинг выполняется с помощью cheerio:
- Извлечение текста из .tgme_widget_message_text
- Извлечение изображений из .tgme_widget_message_photo_wrap
- Извлечение просмотров из .tgme_widget_message_views

5.5.2. TelegramService

Сервис для работы с Telegram Bot API:

- getChannelPosts(limit) - получение сообщений
- getMessageById(messageId) - сообщение по ID
- getFileUrl(fileId) - URL файла
- checkBotAvailability() - проверка бота
- getChannelInfo() - информация о канале

5.5.3. TelegramSyncService

Сервис автоматической синхронизации:

- sync() - синхронизация новых сообщений
- syncWithRetry() - синхронизация с повторами
- processMessage() - обработка сообщения

Cron задача: запуск каждые 5 минут (300 секунд)

8.1. Итоги практики

В ходе практики было разработано полнофункциональное веб-приложение для 
СДЮШОР "Олимпиец" со следующим функционалом:

1. Публикация новостей с автоматической синхронизацией из Telegram
2. Система полнотекстового поиска по контенту
3. Расширенные возможности доступности для людей с ОВЗ
4. Адаптивный дизайн для всех устройств
5. Тёмная/светлая тема оформления
6. AI-функции для сокращения текста

8.2. Приобретённые навыки

- Разработка на Next.js 16 с App Router
- Работа с React 19 и современными хуками
- State management с Zustand
- Серверное состояние с TanStack Query
- Стилизация с Tailwind CSS 4
- Создание доступных интерфейсов (WCAG)
- Разработка REST API на NestJS
- Работа с PostgreSQL и TypeORM
- Интеграция со сторонними API (Telegram)
- Контейнеризация с Docker
- Написание документации компонентов (Storybook)

8.3. Технические достижения

- Использование современных версий фреймворков (Next.js 16, React 19)
